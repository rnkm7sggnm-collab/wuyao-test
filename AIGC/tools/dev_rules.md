# Development Rules

## Principles

- Build the simplest version that solves the actual task.
- Keep implementation clear, local, and easy to change.
- Avoid unnecessary frameworks, abstractions, or automation.
- Prefer readable names and obvious file structure.
- For prototypes, make the first screen usable immediately.

## Working Format

- Goal: what the tool should do.
- Inputs and outputs: define them clearly.
- Implementation: small steps with visible results.
- Next steps: only the most useful improvements.
