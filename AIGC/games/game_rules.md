# Game Rules

## Principles

- Start from the player action, not the lore.
- Keep prototypes small and playable.
- Define the core loop before adding systems.
- Use existing engines or libraries when they make development faster.
- Avoid over-designing mechanics before testing the feel.

## Working Format

- Pitch: one sentence.
- Core loop: action, feedback, reward.
- Rules: only what is needed to play.
- Prototype plan: smallest testable version.
